# 網頁使用方式

## 網站網址
http://securitylab.cyou/

## 開始使用
1. 若尚未登入將會被導至登入畫面，無帳號的話請先註冊
2. 若是已經登入過了，將會被導至主頁面

## 去識別化方法介紹

### DPView
擅長處理資料筆數較多的大型資料集，文字欄位會自動處理，使用時只需要設定數字欄位的意義即可

### k_Anonymity
當檔案的筆數較少時，能夠快速地進行去識別化，需要設定文字之間的關聯以做去識別化，但 k_Anonymity 無法處理帶有空白的資料，如需處理帶有空白欄位的資料，請使用 DPView

## 資料集限制
1. 檔案需存為 .csv 檔案 (逗號分隔檔案，欄位需以逗號分隔)
2. 資料中不能存在半形逗號，因為將會與分隔符號混淆
3. 單一欄位 domain 大小不能超過 50
4. 檔案大小不能超過 4 MB
5. 請先刪除具識別性的欄位，如 ID、姓名
6. 如果檔案內容並非英文語系，請將檔案編碼存為 UTF-8
7. 資料集內容不能擁有時序關係，因為去識別化完成後，將會打亂所有順序
8. 要公布的內容請放在最後一行(column)，只有最後一行不會進行去識別化，例如薪資表的薪水欄位、病歷表的疾病名稱等

## 執行去識別化
1. 執行時間與檔案的 domain 大小有關，因此可能出現小檔案執行很久，但大檔案執行很快的情形
2. 執行過久的時候，可直接關閉網頁，程式將於背景繼續執行，下一次進入首頁時，若尚未執行完畢，將會詢問是否要查看目前的執行進度，若已經執行完畢，則會詢問是否要查看去識別化結果
3. 同時間只能執行一個去識別化，如果前一個去識別化尚未執行完畢，可以選擇放棄前一個去識別化，以執行當前想執行的去識別化

## 可用性檢測
去識別化執行完成後，可以藉由機器學習來檢測資料的可用性，將會把最後一行(column)當作預測目標，以其他欄位作為特徵來預測，首先請根據預測目標的內容，以選擇模型是回歸模型或是分類模型，再選擇想使用的機器學習方法
